# Semantic HTML Project

## Overview
This project demonstrates the use of semantic HTML elements to create a well-structured web page. By utilizing elements such as `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, and `<footer>`, the project enhances accessibility and improves search engine optimization (SEO).

## Setup Instructions
1. Clone the repository to your local machine.
2. Open the `index.html` file in your web browser to view the project.
3. Modify the `css/style.css` file to customize the appearance of the web page as needed.

## Semantic HTML Practices
- **Header**: Contains introductory content or navigational links.
- **Nav**: Defines a set of navigation links.
- **Main**: Represents the dominant content of the `<body>`.
- **Section**: Represents a thematic grouping of content, typically with a heading.
- **Article**: Represents a self-contained composition that could be distributed independently.
- **Footer**: Contains information about its containing element, such as copyright or contact information.

This project serves as a foundation for understanding and implementing semantic HTML in web development.


# README.md

# Projekt Semantic HTML

## Przegląd
Ten projekt demonstruje wykorzystanie semantycznych elementów HTML do tworzenia dobrze ustrukturyzowanej strony internetowej. Dzięki użyciu takich elementów jak `<header>`, `<nav>`, `<main>`, `<section>`, `<article>` i `<footer>`, projekt zwiększa dostępność oraz poprawia optymalizację pod kątem wyszukiwarek (SEO).

## Praktyki Semantic HTML
- **Header**: Zawiera treści wprowadzające lub linki nawigacyjne.
- **Nav**: Definiuje zestaw linków nawigacyjnych.
- **Main**: Reprezentuje główną treść elementu `<body>`.
- **Section**: Reprezentuje tematyczną grupę treści, zazwyczaj z nagłówkiem.
- **Article**: Reprezentuje samodzielną kompozycję, która może być dystrybuowana niezależnie.
- **Footer**: Zawiera informacje o swoim elemencie nadrzędnym, takie jak prawa autorskie lub dane kontaktowe.
